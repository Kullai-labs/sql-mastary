SQL Portfolio & Resume Guide

1. Listing SQL Skills on Your Resume

When applying to MNCs, don't just list "SQL" under your skills. Categorize it to show depth:

Basic/Intermediate

Joins, Aggregations, Subqueries, Group By, Filtering.

Advanced Analytics

Window Functions, CTEs, Cohort Analysis, Recursive Queries.

Database Optimization

Indexing (B-Tree, Hash), Query Execution Plans, Partitioning.

2. Case Study Resume Bullet Points

Copy and customize these high-impact description lines for your projects section:

Swiggy Analytics Project
"Designed and implemented a relational schema for a mock Swiggy database; wrote complex multi-table JOINs and GROUP BY aggregation scripts to calculate restaurant revenues, user order frequencies, and average spending habits."
Rapido Driver Metrics
"Engineered SQL queries incorporating CASE logic and conditional filtering to aggregate Rapido ride metrics, quantifying drivers' trip completion percentages, cancellations, and average fare metrics per vehicle type."
Amazon Inventory Monitoring
"Constructed database schemas utilizing integrity constraints and CHECK directives to model Amazon sales; wrote alert scripts to identify stock shortages and report total revenues aggregated by brand names."
Spotify Streaming Insights
"Created streaming log analytics templates tracking songs play counts and artist streams; utilized sorting and limit partitions to identify top-performing audio content."

3. Submitting to GitHub Portfolio

Follow these steps to submit your completed work to GitHub and showcase it to recruiters:

  • Create a new GitHub repository named sql-portfolio.
  • Upload the case-study SQL scripts from the /projects folder in this repository.
  • Add a detailed README.md explaining the problem, the database schema, and listing the queries.
  • Link your GitHub repository in your resume portfolio section.
